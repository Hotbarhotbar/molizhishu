## 工作流设计

### 主工作流：选题扫描与推送

```
[触发] → [并行扫描] → [合并去重] → [规则评估] → [生成Brief] → [飞书推送]
```

### 节点详细设计

#### 节点1：触发（Trigger）

- **定时触发**：
  - 早间：工作日 9:27，全量扫描
  - 午后：工作日 14:13，平台动态聚焦
- **手动触发**：飞书群聊 @Bot "找选题" / "今天有什么热点"

#### 节点2：并行扫描（Parallel Search）

6个搜索分支并行执行：

| 分支 | 平台 | 搜索方向 | 目标条数 |
|------|------|---------|:------:|
| 分支1 | 微博热搜 | AI工具、品牌营销、搜索引擎变化 | 3-5 |
| 分支2 | 知乎热榜 | AI、营销、数字化转型 | 3-5 |
| 分支3 | 百度风云榜 | 企业服务、科技、AI | 3-5 |
| 分支4 | 36氪/虎嗅 | AI创投、企业服务融资 | 3-5 |
| 分支5 | 微信搜一搜 | GEO、AI搜索、品牌监测 | 3-5 |
| 分支6 | 小红书 | AI工具推荐、品牌运营 | 2-3 |

每个分支使用对应平台的关键词模板（从知识库加载）。

#### 节点3：合并去重（Merge & Deduplicate）

- 合并6个分支的搜索结果
- 同一事件跨平台出现 → 合并为一条，标注多平台来源
- 去重后目标：10-15条候选

#### 节点4：规则评估（Evaluate）

**输入**：去重后的候选列表 + Coze 知识库（同步自 AI_Writing_Vault）

**评估逻辑（Prompt 节点）**：

```
你是一个GEO行业公众号的选题编辑。请对以下热点候选进行三维评估：

## 知识库规则（从知识库加载）
{brand_voice_doc}
{evaluation_rubric}
{recent_articles}

## 候选列表
{raw_candidates}

## 评估步骤
1. **硬过滤**：
   - 中国市场优先（国内话题≥70%，海外限3条）
   - 必须能自然关联产品能力（AI答案监测/品牌体检/GEO诊断/竞品对比/报告复盘/风险预警）
   - 拒绝纯娱乐八卦、与AI/品牌/营销无关的话题

2. **三维打分**（每维1-5分）：
   - 用户价值（35%）：是否打到服务商痛点？
   - 差异化（35%）：角度是否与竞品/近期文章不同？
   - 传播性（30%）：服务商创始人会不会转发？

3. **分级**：
   - ≥12分 → 推荐写
   - 9-11分 → 可储备
   - <9分 → 放弃

4. **结构化输出**：
   对每个推荐选题，输出：
   - 文章类型 / 第一读者 / 核心矛盾 / 事件事实 / 事实口径
   - 服务商痛点 / 产品能力 / 建议标题 / 最大风险
   - 预估字数 / 建议结构

输出格式：JSON数组，每个选题一个对象。
```

#### 节点5：生成Brief（Generate Brief）

将评估结果格式化为完整的19字段选题Brief（Markdown格式），参考 `AI_Writing_Vault/30_模板/文章Brief模板.md`。

#### 节点6：飞书推送（Feishu Push）

推送飞书卡片消息，包含：

```
┌─────────────────────────────────┐
│  📰 模力指数选题日报              │
│  2026-06-24（周二）              │
│                                  │
│  扫描7个平台，获取15条候选         │
│  通过评估5条，其中：              │
│  🔥 推荐写（≥12分）：3条          │
│  💾 可储备（9-11分）：2条         │
│                                  │
│  ── 推荐选题 ──                  │
│                                  │
│  #1 豆包2.1 Pro发布对GEO的影响    │
│  评分：13/15（价值5 差异4 传播4）│
│  类型：热点解读 | 读者：服务商负责人 │
│  核心矛盾：模型更新了，品牌引用逻辑  │
│  变了，服务商不能等客户来问        │
│  风险：需核验2.1具体能力变化      │
│  [查看详情] [选这个写]            │
│                                  │
│  #2 ...                          │
│  #3 ...                          │
│                                  │
│  ⏰ 下次扫描：明天 9:27           │
└─────────────────────────────────┘
```

## 飞书集成

### 飞书机器人配置

1. 在飞书开放平台创建「模力指数-选题助手」应用
2. 获取 App ID + App Secret
3. 在 Coze「飞书机器人」插件中配置
4. 将 Bot 添加到选题评审群

### 飞书卡片交互

卡片消息支持按钮交互：
- **[选这个写]** → 记录选题确认，通知群内
- **[查看详情]** → 展开完整19字段Brief
- **[换一批]** → 触发重新扫描（展示储备选题）
