### 3.3 合并去重节点

**节点类型**：**代码节点**（Code）

**语言**：Python

**代码**（直接复制）：

```python
import json

def main(args):
    """
    合并6路搜索结果，去重，输出结构化候选列表
    """
    # 收集所有搜索结果
    all_results = []
    
    # 从6个搜索节点获取结果
    for key, value in args.items():
        if key.startswith('search_') and value:
            try:
                if isinstance(value, str):
                    items = json.loads(value)
                elif isinstance(value, list):
                    items = value
                else:
                    continue
                    
                for item in items:
                    title = item.get('title', '') or item.get('name', '')
                    snippet = item.get('snippet', '') or item.get('description', '') or item.get('summary', '')
                    url = item.get('url', '') or item.get('link', '')
                    
                    if not title:
                        continue
                        
                    all_results.append({
                        'title': title.strip(),
                        'snippet': snippet.strip()[:300] if snippet else '',
                        'url': url.strip() if url else '',
                        'source': key.replace('search_', ''),
                    })
            except:
                continue
    
    # 简单去重：标题相似度 > 0.7 视为重复
    seen = []
    deduped = []
    for item in all_results:
        title_lower = item['title'].lower()
        is_dup = False
        for s in seen:
            # 简单基于公共子串的去重
            common_words = set(title_lower.split()) & set(s.split())
            if len(common_words) >= 3:
                is_dup = True
                break
        if not is_dup:
            seen.append(title_lower)
            deduped.append(item)
    
    # 限制数量
    deduped = deduped[:20]
    
    return {
        'candidates': json.dumps(deduped, ensure_ascii=False),
        'total_scanned': len(all_results),
        'total_deduped': len(deduped),
        'sources_covered': list(set(item['source'] for item in deduped))
    }
```

**输入变量映射**：把6个搜索节点的输出分别映射到此节点的 `search_weibo`, `search_zhihu`, `search_baidu`, `search_36kr`, `search_wechat`, `search_xiaohongshu`

### 3.4 规则评估节点（核心）

**节点类型**：**大模型节点**（LLM）

**提示词**（完整复制）：

## 你的知识

{知识库检索结果}

## 今日候选热点

{合并去重节点的 candidates 输出}

## 评估流程（严格按顺序执行）

### 第一步：硬过滤

对每条候选，依次检查：

1. **中国市场优先**：国内话题必须占 >= 70%，海外主题最多保留3条，且仅限 OpenAI/GPT、Anthropic/Claude、Google/Gemini 等头部 AI 大厂的动态，或明确提到 GEO、AI搜索、AI答案引用、广告商业化、信源/抓取争议的新闻。

2. **产品关联硬筛**：每条候选必须能自然关联到至少1个产品能力：
   - AI答案监测 / 品牌体检 / GEO诊断 / 竞品对比 / 信源分析 / 报告复盘 / 风险预警
   - 如果完全无法关联 → 直接放弃（除非是纯行业观察，最多保留1条）

3. **GEO相关性**：话题必须与 AI搜索、品牌可见性、平台政策、广告变现、模型更新、服务商运营相关。纯娱乐八卦、与AI/品牌/营销无关 → 放弃。

### 第二步：三维打分

对通过硬过滤的候选，逐条打分（1-5分）：

**用户价值（权重35%）**
- 5分：直接命中服务商具体痛点（客户不认账/续费被质疑/竞品突然被推荐/AI说错了品牌/投毒风险/报价被压），可立刻转化为客户沟通话术
- 4分：明确涉及经济/交付/技术/竞品/合规/续费压力，但转化话术需要进一步挖掘
- 3分：与服务商痛点相关，但偏行业趋势而非具体压力
- 2分：话题有趣但与服务商日常运营关联弱
- 1分：纯粹行业新闻，服务商无法直接用于获客或交付

**差异化（权重35%）**
- 5分：角度是同行想不到/不敢写的，提供新的判断框架，或挖掘了被忽视的数据
- 4分：角度有差异，不是简单的事件转述，有明确的独立判断
- 3分：角度偏常规但切入点不错，可能会和同行撞题但写法不同
- 2分：大概率被同行覆盖，差异化仅靠不同案例或措辞
- 1分：已被大量覆盖，无法提供新信息或新角度

**传播性（权重30%）**
- 5分：服务商创始人/负责人大概率会转发到群和朋友圈，含具体数字、清晰判断、反常识观点
- 4分：服务商从业者会觉得有用并收藏，有分享动力
- 3分：关注GEO的人会点开看，但转发意愿一般
- 2分：行业内可能传播，但缺乏转发引爆点
- 1分：缺乏传播动力

**加权总分 = 用户价值×0.35 + 差异化×0.35 + 传播性×0.30**

### 第三步：分级

- 总分 >= 12：🔥 推荐写（立即推送）
- 总分 9-11：💾 可储备
- 总分 < 9：❌ 放弃

### 第四步：对推荐选题（>=12分）进行深度分析

逐一回答以下8个问题：
1. 这件事让客户多花了什么钱？
2. 这件事让服务商在哪个环节被质疑？
3. 如果没有监测，双方会靠什么争论？
4. 哪个指标能把争论变成事实？
5. 竞品在这里制造了什么压力？
6. 有没有投毒、虚假、低质内容或合规风险？
7. 这件事影响售前、交付、验收还是续费？
8. 文章里的需求承接的是体检、监测、诊断、报告，还是风险预警？

## 第四步：配置定时任务（5分钟）

Coze 平台支持两种定时方式：

### 方式A：Coze 内置定时触发器（推荐）

1. 工作流编辑器 → 将「开始节点」的触发方式改为 **定时触发**
2. 添加两个定时规则：

| 定时规则 | Cron表达式 | 说明 |
|----------|------------|------|
| 早间扫描 | `27 9 * * 1-5` | 工作日9:27，全量扫描 |
| 午后扫描 | `13 14 * * 1-5` | 工作日14:13，平台动态聚焦 |

3. 为「午后扫描」创建一个工作流副本，将 `focus` 参数默认值改为 `platform`

### 方式B：Coze 定时任务（如果内置触发器不可用）

1. Coze 左侧导航 → **定时任务** → **新建**
2. 选择 Bot「模力指数-选题助手」
3. 设置触发时间（同上表）
4. 输入触发指令：`执行选题扫描` 或直接调用工作流

## 第五步：配置手动触发（群聊@Bot）

## 第七步：调优（持续）

### 需要观察的指标

| 指标 | 观察方法 | 优化方向 |
|------|----------|----------|
| 选题命中率 | 人工通过的选题数 / AI推荐的选题数 | 调整评分阈值 |
| 搜索覆盖率 | 扫描结果是否漏掉了明显热点 | 补充搜索关键词、增加平台 |
| 评估准确性 | AI给的分数是否符合人工判断 | 调优评估Prompt、补充知识库案例 |
| 飞书推送成功率 | 卡片是否正常到达 | 检查飞书插件配置 |
| 定时任务稳定性 | 是否按时触发 | 检查Coze定时任务日志 |

### 常见问题排查

| 问题 | 可能原因 | 解决方法 |
|------|----------|----------|
| 搜索节点无结果 | 搜索词太窄/平台限制 | 放宽搜索词、更换搜索插件 |
| LLM评估过于保守 | 评分Prompt阈值太高 | 在知识库中补充更多「好选题」案例 |
| LLM评估过于激进 | 评分Prompt阈值太低 | 提高硬过滤标准 |
| 飞书卡片格式错乱 | JSON变量未替换 | 在生成Brief节点中直接生成完整卡片JSON |
| 知识库检索不生效 | 分块方式不对 | 改为「按文档分块」或调整最大召回数 |
