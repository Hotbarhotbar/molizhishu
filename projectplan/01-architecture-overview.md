# 模力指数公众号AI化 — 架构总览

## 背景

模力科技（GEO品牌AI监测平台「模力指数」）通过微信公众号文章营销获客。目前文章创作依赖 Claude Code 内的 `geo-content-writer` Skill（7阶段全流程），从选题到发布均由人工驱动。

目标：从选题环节开始，逐步构建 AI 自动化工作流，最终实现"AI客户经理 + 工人Agent"的全流程AI化内容工厂。

> **注意**：Coze 替代方案正在讨论中，详见 [06-coze-replacement-analysis.md](06-coze-replacement-analysis.md)。当前推荐 Phase 1 用 Claude Code 原生能力替代 Coze，Phase 3+ 视需求评估 Dify。

## 三层架构

```
┌─────── 工作流编排层（Coze / Claude Code）──────┐
│                                            │
│  定时触发 / 手动触发                        │
│       ↓                                    │
│  热点扫描（7平台）                          │
│       ↓                                    │
│  知识库规则评估+三维打分                    │
│       ↓                                    │
│  生成选题简报（19字段完整Brief）            │
│       ↓                                    │
│  飞书机器人 → 推送到群聊                    │
│                                            │
└────────────────┬───────────────────────────┘
                 │
                 ↓
    ┌────────────────────────────┐
    │     飞书群聊（人工决策层）   │
    │                             │
    │  看到3-5个选题卡片           │
    │  老板/编辑讨论、打分         │
    │  确定今天写哪个              │
    │                             │
    └────────┬───────────────────┘
             │  "写选题#2，仿照那篇ChatGPT的"
             ↓
    ┌────────────────────────────────────┐
    │   Claude Code（深度创作执行层）      │
    │                                     │
    │  /geo-analyze → 参考文章风格分析     │
    │  /geo-write   → 多模式写作           │
    │  /geo-humanize → 反AI检测人味化      │
    │  /geo-publish → 排版+发布到草稿箱    │
    │                                     │
    └────────────────────────────────────┘
```

### 各层职责

| 层 | 平台 | 职责 | 当前自动化程度 |
|----|------|------|:---:|
| **工作流层** | Coze（扣子） | 选题扫描、规则评估、Brief生成、飞书推送 | 全自动 |
| **决策层** | 飞书群聊 | 人工评审选题、确定方向、分派写作任务 | 人工 |
| **执行层** | Claude Code | 风格分析、深度写作、人味化改写、排版发布 | 人工调用 |

## 关键设计原则

1. **人工节点可插拔**：飞书审批环节在 Coze 工作流中设计为可旁路节点，后续AI化程度提高后可逐步后撤
2. **文件契约为交接面**：层与层之间通过 `topic-brief.md`（Markdown文件）交接，不绑定任何平台
3. **知识库为单一真相源**：`AI_Writing_Vault/` 是品牌声音、写作规则、评分标准的唯一来源，Coze 知识库需与之同步
4. **渐进式演进**：每个阶段独立可用，不依赖后续阶段的完成

## 文章创作流程（现状 vs 目标）

```
现状：
  人工发现热点 → 人工判断选题 → Claude Code写稿 → 人工修改 → 人工发布

Phase 1 目标：
  Coze自动选题 → 飞书人工确认 → Claude Code写稿 → 人工修改 → 人工发布

Phase 5 终局：
  Coze客户经理Agent → 工人Agents（选题→写作→修改→评分→发布）→ 飞书异常告警
```
